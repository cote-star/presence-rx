# Presence Rx Marketing Demo Pack

Use these demos for the README, submission video, live walkthrough, social cuts, or a pitch deck. The screenshots and animated WebP are generated from the local webapp and can be refreshed with:

```bash
cd webapp
npm run dev

# in another shell from the repo root
node scripts/capture_demo_assets.mjs
node scripts/record_demo_webp.mjs
```

## Animated WebP Walkthrough

The README hero uses [presence-rx-walkthrough.webp](demo-assets/presence-rx-walkthrough.webp). It slowly scrolls through:

1. Diagnosis and gap classification
2. Strategic actions by gap type
3. Channel activation and audiences
4. Blocked claim with safe rewrite
5. Future-direction preview marked as modeled

Keep this asset short and quiet. It is meant to show the product flow at a glance, not replace the narrated submission video.

## Demo 1: Gap Analysis

Screenshot: [gap-analysis.png](demo-assets/gap-analysis.png)

### 10-Second Cut

**Show:** Conversation Blocks.

**Say:** "Presence Rx does not just report low visibility. It classifies the type of gap. For Nothing Phone, Smartphone Design is a stronghold, Mobile Ecosystem is discovery, Minimalist Hardware is perception, and Wireless Audio is attention."

**Point at:** the gap pills, competitor owner, signal alignment, and Gemini perception panel.

### 25-Second Product Walk

**Show:** top two cards on the Diagnosis page.

**Say:** "The important distinction is that these are not the same marketing problem. Smartphone Design is a benchmark to defend. Mobile Ecosystem is not about claiming Apple-level platform dominance; it is about making the cohesive product-family story discoverable. That is why the next move is schema, canonical source pages, and new Peec monitoring prompts."

**Point at:** Action Priority, Recommended next move, Strategic Context.

### 45-Second Judge Version

**Show:** Diagnosis page, then scroll through the first three topic cards.

**Say:** "Peec is the source of truth for visibility. Presence Rx adds classification. Each topic is scored as a stronghold, perception gap, discovery gap, or attention gap. That matters because the wrong diagnosis produces the wrong action. If Minimalist Hardware is a perception gap, the fix is explicit trait-language proof. If Wireless Audio is an attention gap, the fix is more sustained public signal. The product turns visibility into a decision, not just a chart."

## Demo 3: Channel Activation

### 10-Second Cut

**Show:** Where to Engage.

**Say:** "Presence Rx maps each gap type to channels. Editorial, UGC, owned, YouTube, Reddit, and Peec monitoring all have a role depending on the issue."

**Point at:** channel cards and linked topic counts.

### 25-Second Product Walk

**Show:** Channel Activation section.

**Say:** "This is where the action becomes operational. Owned content is best for discovery and perception. Editorial is best for perception and attention. UGC and Reddit are useful when the category signal is too thin. The channel plan is based on the issue type, not a generic media checklist."

**Point at:** Best for labels and Who to Reach.

### 45-Second Judge Version

**Show:** Channel cards, then audience segment cards.

**Say:** "Once Presence Rx knows the issue type, it can choose where to act. Minimalist Hardware needs design press and owned positioning content. Mobile Ecosystem needs canonical pages and structured FAQ content. Wireless Audio needs sustained creator and community proof. The same dashboard also maps the audiences that should receive that work: design-conscious buyers, tech enthusiasts, Android switchers, and minimalism seekers."

### 25-Second Product Walk

**Show:** Claims & Simulator page.

**Say:** "The claim simulator checks marketing language against the evidence ledger. It blocks ownership overclaims, explains the visibility gap, and provides a safer rewrite the marketer can use today."

**Point at:** Matched topic, competitor owner, safe rewrite.

### 45-Second Judge Version

**Show:** blocked claims table, then the interactive simulator result.

## Recording Notes

- Keep the first demo under 90 seconds.
- Lead with the four-feature spine: gap, action, activation, claim.
- Use the claim simulator as the emotional peak, not the whole story.
- Say "modeled preview" when showing the Future page.
- Say "Peec is the source of truth for visibility" whenever metrics are discussed.
